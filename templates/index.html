Quick Start Guide

Welcome to the Wind Tunnel Control System. This guide will help you get started.

Basic Operation

  1. View Sensor Data: Real-time sensor readings appear as cards on the main dashboard
  2. Control Mode: Choose between PID (airspeed control) or Manual (direct fan speed) using the toggle
  3. Start Control: Set your target and click Start
  4. Emergency Stop: Click the red stop button in the top bar to immediately stop the fan

Interface Overview

  • Status Bar (Top): Connection status, WiFi indicator, and control buttons
  • Sensor Cards: Live data display with graphs and history
  • Control Panel: Switch between PID and manual fan control
  • Settings: Configure sensors, PID parameters, and export data

PID Airspeed Control

The PID controller automatically adjusts fan speed to maintain a target airspeed.

Using PID Control

  1. Ensure the mode toggle is set to PID mode (toggle OFF/left position)
  2. Set your target airspeed using the slider (0-30 m/s)
  3. Click Start to begin control
  4. Monitor the Target, Actual, and Fan values
  5. The controller will automatically adjust fan speed to reach and maintain the target

Auto-Tune Feature

Auto-tune automatically determines optimal PID parameters:

  1. Go to Settings â†’ PID Control tab
  2. Click Start Auto-Tune
  3. Set a test setpoint (typically 10-15 m/s recommended)
  4. Wait for the process to complete (typically 2-3 minutes)
  5. The system will automatically apply the tuned parameters

Manual Tuning

Advanced users can manually adjust PID parameters in Settings:

  • Kp (Proportional): Immediate response to error (typical: 1-3)
  • Ki (Integral): Eliminates steady-state error (typical: 2-5)
  • Kd (Derivative): Reduces overshoot (typical: 0.5-2)
  • Min Fan Speed: Minimum output when control is active (typical: 15-20%)
ðŸ’¡ Tip:

The system includes setpoint rate limiting (3 m/s per second) and anti-windup protection to prevent overshoot and oscillations.

Sensor Configuration

The system supports multiple sensor types for comprehensive data acquisition.

Supported Sensor Types

  • UDP Network Sensors: ESP32-based wireless sensors (auto-detected)
  • I2C Sensors: BMP280, BME280, SDP8xx, MPU6050, INA219, VL53L0X
  • ADC Sensors: Analog sensors via ADS1115 or MCP3008
  • GPIO Sensors: Digital sensors, DHT11/22
  • 1-Wire Sensors: DS18B20 temperature sensors
  • Calculated Sensors: Formula-based virtual sensors

Adding Sensors

  1. Go to Settings â†’ Sensors tab
  2. Click + Add Sensor
  3. Select sensor type and configure parameters
  4. UDP sensors are automatically created when they start sending data

Calculated Sensors

Create virtual sensors using formulas:

  • Reference other sensors by their ID (e.g., lift / drag)
  • Use mathematical functions: sqrt(), abs(), max(), min()
  • Example: sqrt(max(0, differential_pressure * 2 / 1.225))

Data Export

Export sensor data to USB drives for offline analysis.

Exporting Data

  1. Click the Download icon in the top bar
  2. Select time range (all data, last N minutes/hours)
  3. Choose your USB drive from the list
  4. Click Export
  5. Wait for the export to complete

Export Format

Data is exported as CSV files with:

  • Timestamp (ISO 8601 format)
  • All sensor values at each time point
  • Metadata header with sensor information
  • Compatible with Excel, Python, MATLAB, etc.

Troubleshooting

Connection Issues

Status shows "Disconnected":

  • Check that the server is running
  • Verify network connection
  • Refresh the page (F5)
  • Check browser console for errors (F12)

PID Control Issues

PID not starting or showing errors:

  • Ensure airspeed sensor is configured in Settings
  • Check that the sensor is receiving data
  • Verify PID parameters are set (run auto-tune if needed)

PID oscillating or overshooting:

  • Try running auto-tune again
  • Manually reduce Kp (proportional gain)
  • Increase Kd (derivative gain) to dampen oscillations

Sensor Issues

UDP sensors not appearing:

  • Check ESP32 is powered on and connected to network
  • Verify UDP port matches (default: 5005, 5006, 5007)
  • Check firewall settings
  • Look for "UDP received" messages in server console

Sensor showing "Error" or no data:

  • Check hardware connections
  • Verify I2C address is correct
  • Check server console for error messages
  • Try disabling and re-enabling the sensor
